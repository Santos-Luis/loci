**Date:** 2026-05-30  
**Status:** Approved

## Context

### Developer background

Luis is an experienced TypeScript/Node.js engineer. His existing open-source projects include:

- **taskdeck** — a local-first cron job manager built with Fastify + React + SQLite + Vite. This is the closest architectural reference: the data model, repository pattern, and React frontend conventions from taskdeck should be followed wherever possible in this project.
- **santosclaw** — a personal AI assistant that runs on a cron schedule, calls the Claude Code CLI as a subprocess, and delivers daily news digests and learning modules via Telegram. This project proves the Claude CLI subprocess pattern works well in practice.
- **news-digest** — a static Astro site that archives AI-generated daily digests to GitHub Pages, generated by santosclaw.

### Why this project

santosclaw works well as a personal tool but is not open-sourceable — it is deeply tied to Luis's Telegram account, hardcoded topics, and personal learning goals. The goal here is to build a **generalised, open-source version** of the same idea: a persistent personal knowledge agent that anyone can run locally, configure to their own interests, and own entirely.

### Hard constraints

- **No cloud hosting, no server** — the app must run entirely on the user's machine. No storage costs, no hosting fees. A local web server (localhost) is acceptable only if it's part of the desktop app process, not a separately deployed service.
- **No Anthropic API key required** — the app uses the Claude Code CLI (`claude`) as a subprocess. Users need Claude Code installed and authenticated (Pro subscription). This means no per-token billing and no API key management.
- **Open source** — MIT licence. Must work out of the box for any developer who has Claude Code CLI installed.

### What "agentic" means here

The app is both **reactive** (the user asks questions and gets answers grounded in their accumulated memory) and **proactive** (a background agent autonomously generates summaries, cross-topic connections, and thought-provoking questions on a schedule, without the user asking). The agentic behaviour is the proactive layer — it acts on its own, builds up the knowledge base over time, and surfaces things the user didn't know to ask for.

## Overview

A local-first desktop application that acts as a personal knowledge assistant. It is both reactive (answer questions using accumulated memory) and proactive (autonomously generate insights, summaries, and questions in the background). All data stays on the user's machine. No cloud hosting. No API key required — uses the Claude Code CLI.

## Architecture

```
Electron Main Process (Node.js)
├── SQLite database          ← all persistent state
├── Claude CLI client        ← spawns `claude` subprocess
├── Background agent         ← node-cron, proactive tasks
└── IPC handlers             ← bridge to renderer

Electron Renderer (React + Vite)
├── Dashboard
├── Ask (Q&A chat)
├── Topics
├── Notes
├── Insights
└── Settings
```

The main process owns all data access and Claude calls. The renderer communicates exclusively via Electron IPC — no HTTP server, no exposed ports. The app is packaged with `electron-builder` (`.dmg` on macOS).

**Tech stack:** Electron · Vite · React 19 · TypeScript · better-sqlite3 · node-cron · child_process

**Prerequisite for users:** Claude Code CLI installed and authenticated.

## Data Model

Six SQLite tables:

```sql
topics        (id, name, description, created_at)
notes         (id, topic_id NULLABLE, title, content, created_at, updated_at)
conversations (id, topic_id NULLABLE, title, created_at)
messages      (id, conversation_id, role TEXT CHECK(role IN ('user','assistant')), content, created_at)
insights      (id, topic_id NULLABLE, type TEXT CHECK(type IN ('summary','connection','question')), content, generated_at)
settings      (key TEXT PRIMARY KEY, value TEXT)
```

FTS5 virtual tables shadow `notes`, `messages`, and `insights` for BM25-scored full-text search.

`topics` are user-defined areas of interest. `notes` and `conversations` are optionally attached to a topic. `insights` are the proactive agent's output. `settings` holds configuration (Claude CLI path, model, schedule, data directory).

## Memory & Retrieval

Memory is built at query time — no embeddings, no vector DB.

For each user message:

1. FTS5 BM25 search across notes, messages, and insights scored by relevance to the input — top 10 results
2. Last 5 messages from the current conversation (continuity)
3. Combined and injected as context into the system prompt before the Claude call

This gives intelligent retrieval at sub-millisecond cost with no extra LLM call.

---

## Reactive Layer (Q&A)

Flow when the user sends a message in Ask:

1. FTS5 retrieval (step above)
2. Build system prompt: topics list + retrieved context + conversation history
3. Spawn `claude -p "<full prompt>" --output-format stream-json --print`
4. Stream tokens back to renderer via IPC as they arrive
5. On completion, persist the full exchange to `messages` table

Streaming via `--output-format stream-json` ensures the response feels immediate rather than arriving all at once after a multi-second wait.

## Proactive Agent (Background)

`node-cron` runs inside the main process. Two scheduled jobs:

### Daily Summary (default: 8:00am, configurable)

For each topic with activity in the last 7 days:

- Fetches recent notes and conversation messages for that topic
- Spawns a Claude call asking for a concise synthesis
- Saves result as an `insight` of type `summary`
- Fires an Electron `Notification` when complete

### Weekly Synthesis (default: Friday 5:00pm, configurable)

- Fetches all recent topic summaries
- Spawns a single Claude call to find cross-topic connections and surface 2–3 thought-provoking questions
- Saves results as `connection` and `question` insights
- Fires an Electron `Notification` when complete

Both jobs run non-blocking in the background. The Insights view shows a badge when new insights are unread. Jobs can be disabled entirely from Settings.

**Notifications** use Electron's built-in `Notification` API from the main process — works on macOS and Windows, no extra dependency.

## UI & Navigation

Left sidebar with six views:

### Dashboard

- Latest unread insight
- Quick-ask input (opens a new conversation in Ask)
- Recent activity: last 3 conversations, last note edited

### Ask

- Full chat interface, conversations auto-saved
- Optional topic tag per conversation
- Collapsible context panel showing which memory was injected for each exchange

### Topics

- List of user-defined topics
- Per-topic view: attached notes, tagged conversations, generated insights
- Creating a topic requires only a name (description optional)

### Notes

- Freeform markdown editor
- Notes are free-floating or attached to a topic
- Full-text searchable via FTS5

### Insights

- All proactive agent output in reverse-chronological order
- Filterable by type: summary / connection / question
- Filterable by topic
- Badge in sidebar nav when new insights arrive

### Settings

- Claude CLI binary path
- Model selection (`sonnet` / `haiku`)
- Daily job: on/off + time
- Weekly job: on/off + day + time
- Data directory location

## Open Source Considerations

- Users must have Claude Code CLI installed (documented in README)
- All data stored in a configurable local directory (default `~/.<app-name>/`, named after the repository)
- No telemetry, no analytics, no network calls except the `claude` subprocess
- `.gitignore` excludes the database and settings by default
- MIT license

## Out of Scope (v1)

- Two-pass LLM reranking for memory retrieval (revisit if FTS5 proves insufficient)
- Vector embeddings
- Multi-device sync
- Mobile
- Plugins or extensibility API
